# Implementation Notes: Perf Event Syscall Monitoring

---

## Problem Solved: Struct Alignment (C ↔ Go)

### Issue
C struct `file_read_write_event` had implicit padding for `__u64` alignment, but Go struct didn't account for it. Result: binary deserialization read garbage SIZE values (e.g., 395136991232 instead of 84).

### Root Cause
```c
struct file_read_write_event {
    __u32 pid;       // offset 0-3
    __u32 tid;       // offset 4-7
    __u32 uid;       // offset 8-11
    __u32 gid;       // offset 12-15
    int fd;          // offset 16-19
    // C compiler adds 4 bytes padding here for __u64 alignment!
    __u64 size;      // offset 24-31 (NOT 20!)
    __u64 timestamp; // offset 32-39
};
// Actual size: 40 bytes (not 36!)
```

Go was reading 36 bytes, missing the padding → SIZE read from wrong offset.

### Solution
Add explicit padding field in BOTH C and Go:

**C:**
```c
struct file_read_write_event {
    __u32 pid, tid, uid, gid;
    int fd;
    __u32 pad;       // ← Explicit 4-byte padding
    __u64 size;
    __u64 timestamp;
};
```

**Go:**
```go
type ReadWriteFileEvent struct {
    PID, TID, UID, GID uint32
    FD                 int32
    Pad                uint32  // ← Match C exactly
    SIZE               uint64
    Timestamp          uint64
}
```

Now both are 40 bytes, binary.Read() works correctly.

## Perf Event Arrays: How It Works

### BPF Side (C)
```c
struct {
    __uint(type, BPF_MAP_TYPE_PERF_EVENT_ARRAY);
    __uint(key_size, sizeof(__u32));  // CPU ID (automatic)
} open_events SEC(".maps");

SEC("tracepoint/syscalls/sys_enter_openat")
int handle_sys_open(struct trace_event_raw_sys_enter *ctx) {
    struct file_open_event ev = {...};
    bpf_perf_event_output(ctx, &open_events, BPF_F_CURRENT_CPU, &ev, sizeof(ev));
    return 0;
}
```

**Key:** `BPF_F_CURRENT_CPU` automatically routes event to the CPU the syscall ran on (no manual key).

### Go Side
```go
reader, _ := perf.NewReader(objs.Maps.OpenEvents, 4096)  // 4096 = buffer per CPU

for {
    record, _ := reader.Read()
    var event OpenFileEvent
    binary.Read(bytes.NewReader(record.RawSample), binary.LittleEndian, &event)
    // Process event
}
```

### Why Perf Arrays vs Ringbuf?

| Feature | Perf Events | Ringbuf |
|---------|------------|---------|
| Per-CPU routing | Automatic (`BPF_F_CURRENT_CPU`) | Manual |
| Struct serialization | Fixed binary only | Any format |
| Data transport | Per-CPU ringbuffers | Single shared ringbuffer |
| Kernel requirement | 5.0+ | 5.8+ |
| Use case | Simple syscall tracing | Complex event formats |

**For this lab:** Perf events perfect — no serialization needed, per-CPU automatic.

## Tracepoint Context: sys_enter_* Structure

### What is `struct trace_event_raw_sys_enter`?

Kernel-generated structure with syscall arguments:
```c
struct trace_event_raw_sys_enter {
    // (auto-generated by kernel tracepoint)
    __u64 args[6];  // syscall arguments (up to 6)
};
```

### Reading Syscall Arguments

**For openat:**
```c
// int openat(int dirfd, const char *pathname, int flags, int mode)
int dfd = ctx->args[0];              // dirfd
const char *pathname = (const char *)ctx->args[1];  // user-space pointer!
int flags = ctx->args[2];
int mode = ctx->args[3];
```

**Key:** args[1] points to USER-SPACE memory (filename string), so must use `bpf_probe_read_user_str()` not `bpf_probe_read_kernel_str()`.

## Signal Handling: Clean Shutdown with Blocked I/O

### Problem
`reader.Read()` is a blocking syscall that doesn't respect context cancellation. Ctrl+C doesn't exit cleanly.

### Solution
Close readers before canceling context:

```go
ctx, cancel := context.WithCancel(context.Background())

sigChan := make(chan os.Signal, 1)
signal.Notify(sigChan, syscall.SIGINT)
go func() {
    <-sigChan
    fmt.Println("\nShutting down...")
    
    // Close all readers FIRST (unblocks Read() calls)
    oe_reader.Close()
    ce_reader.Close()
    rf_reader.Close()
    wf_reader.Close()
    
    // THEN cancel context
    cancel()
    os.Exit(0)
}()
```

**Why?** Closing the underlying map/reader FD forces Read() to return with EOF, unblocking goroutines.
